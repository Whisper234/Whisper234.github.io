2.8 CSS3 2D转换

转换是 CSS3 的特征之一，可以实现元素的位移、旋转、缩放等效果

  • 移动：translate
  • 旋转：rotate
  • 缩放：scale

2.8.1 2D转换之移动 translate

2D移动是2D转换里面的一种功能，可以改变元素在页面中的位置，类似定位

2.8.1.2 重点

  • translate中的百分比单位是相对于自身元素的translate:(50%,50%)
  • 对行内标签没有效果

2D旋转指的是让元素在二维平面内顺时针旋转或者逆时针旋转

2.8.2.2 重点

  • 角度为正时，顺时针，反之则为逆时针shen
  • 默认旋转的中心点是元素的中心点

2.8.3.1 语法

2.8.3.2 重点

  • x y 默认转换的中心点是元素的中心点（50% 50%）

2.8.4 2D 转换之缩放 scale

2.8.3.1 语法

2.8.3.2 重点

  • 注意后面的参数 x 和 y 用逗号隔开
  • transform:scale(1,1); 宽和高都放大一倍，相当于没有放大
  • transform:scale(2,2); 宽和高都放大了两倍
  • transform:scale(2); 只写第一个参数，第二个参数和第一个参数一样
  • transform:scale(0.5,0.5); 缩小
  • scale的优点，可以设置转换中心点缩放，默认以元素的中心点，且不影响其他盒子

2.8.5 2D 转换综合写法

注意

  1. 同时使用多个转换，其格式为：transform：translate(x,y) rotate(度数) scale(x,y);
  2. 其顺序会影响转换的效果。（先旋转会改变坐标轴方向）
  3. 同时有位移和其他属性时，位移放在最前面

2.8.6 2D 转换总结

  • 2D 移动translate(x, y) 最大的优势是不影响其他盒子，里面参数用% ,是相对于自身宽度和高度来计算的，可以分开写比如 translateX(x)和translateY(y)
  • 2D旋转rotate(度数)，可以实现旋转元素度数的单位是deg
  • 2D 缩放sacle(x,y)里面参数是数字不跟单位可以是小数，最大的优势不影响其他盒子
  • 设置转换中心点 transform-origin:xy;参数可以百分比、 像素或者是方位名词
  • 当我们进行综合写法 ,同时有位移和其他属性的时候,记得要将位移放到最前

2.10 CSS3 3D 转换

特点

  • 近大远小
  • 物体后面遮挡不可见

2.10.1 三维坐标系

三维坐标系其实就是指立体空间,立体空间是由3个轴共同组成的。

  • x轴：水平向右 注意：x右边是正值，左边是负值
  • y轴：垂直向下 注意：y下面是正值,上面是负值
  • z轴：垂直屏幕 注意：往外面是正值,往里面是负值

2.10.2 3D移动 translate 3d

3D移动在2D移动的基础上多加了一个可以移动的方向,就是z轴方向。

  • translform:translateX(100px) :仅仅是在x轴上移动
  • translform:translateY(100px) :仅仅是在Y轴上移动
  • translform:translateZ(100px) :仅仅是在Z轴上移动 ( 注意: translateZ一般用px单位 )
  • transform:translate3d(x,y,z) :其中x、y. z分别指要移动的轴的方向的距离

2.10.3 透视 perspective

在2D平面产生近大远小视觉立体,但是只是效果二维的

  • 透视的单位是像素

透视写在被观察元素的父盒子上面的

  • z轴越大(正值)我们看到的物体就越大。

2.10.4 3D旋转rotate3d

3D旋转指可以让元素在三维平面内沿着x轴, y轴, z轴或者自定义轴进行旋转。

语法

  • transform:rotateX(45deg) : 沿着x轴正方向旋转45deg
  • transform:rotateY(45deg) : 沿着y轴正方向旋转45deg
  • transform:rotateZ(45deg) : 沿着Z轴正方向旋转45deg
  • transform:rotate3d(x,y,z,deg) : 沿着自定义轴旋转deg角度

对于元素旋转的方向的判断我们需要先学习一个左手准则。

左手准则

  • 左手的手拇指指向 x轴的正衣向

  • 其余手指的弯曲方向就是该元素沿着x轴旋转的方向

  1. transform:rotate3d(1,0,0,45deg) 就是沿着x轴旋转45deg
  2. transform:rotate3d(1,1,0,45deg) 就是沿着对角线旋转45deg

2.10.5 3D呈现transfrom-style

  • 控制子元素是否开启三维立体环境
  • transform-style: flat子元素不开启3d立体空间 默认的
  • transform-style: preserve- 3d,子元素开启立体空间
  • 代码写给父级 ,但是影响的是子盒子

  • -ms- :代表ie浏览器私有属性

2.3 结构伪类选择器

结构伪类选择器主要根据文档结构来选择，常用于根据父级选择器里面的子元素

选择符 简介
E:first-child 匹配父元素中的第一个子元素E

​ E:nth-of-type(n) 选择某个父元素的一个或多个特定的子元素

  • n 可以是数字，关键字和公式
  • n 如果是数字，就是选择第n个子元素，里面拿数字从1开始
  • n 可以是关键字：even偶数,odd奇数
  • n 可以是公式：常见的公式如下（如果n是公式，则从9开始计算，但是第0个元素或者超出了元素的个数会被忽略）
公式 取值
2n 偶数
2n+1 奇数
5n 5 10 15 …
n+5 从第5个开始（包含第5个）到最后

区别

E:nth-child 对父元素所有子元素排序选择，先找到第n个，再判断是否和E匹配

E:nth-of-type(n) 对父元素指定子元素排序选择，先判断是否和E匹配，再找到第n个

小结

  • 结构伪类选择器一般用于选择父级里面的第几个孩子
  • E:nth-child 对父元素所有子元素排序选择，先找到第n个，再判断是否和E匹配
  • E:nth-of-type(n) 对父元素指定子元素排序选择，先判断是否和E匹配，再找到第n个
  • 如果是无序列表，用 nth-child
  • 类选择器、属性选择器、伪类选择器，权重为10
  • 关于nth-child（n）n从0开始计算的

color 可选，阴影颜色

CSS浮动

1.浮动

1.1传统网页布局三种方式

  • 普通流（标准流）
  • 浮动
  • 定位

1.2标准流（普通流/文档流）

标签按照规定好的默认方式排列

  1. 块级元素惠独占一行，从下向上顺序排列
    • 常用元素：div、hr、p、h1~h6、ul、ol、dl、form、table
  2. 行内元素会按照顺序，从左向右顺序排列，碰到父元素边缘则自动换行。
    • span、a、i、em等。

标准流是最基本的布局方式

1.3浮动必要性

浮动可以改变元素标签默认的排列方式。

典型应用：可以让多个块级元素一行内排列显示。

多个块级元素纵向排列——标准流

多个块级元素横向排列——浮动

1.4浮动定义

float属性用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘。

语法：

属性值：left（左浮动）、right（右浮动）、none（不浮动）（默认值）。

1.5浮动特性

  1. 浮动元素会脱离标准流（脱标）。
    • 浮动的盒子不再保留原先的位置。
    • 脱离标准流的控制移动到指定位置。
  2. 浮动元素会一行显示并且元素顶部对齐。
    • 如果一行宽度装不下，会另起一行。
  3. 浮动元素会具有行内块元素的特性。
    • 行内元素有了浮动，就不需要转换，可以直接设置宽高。
    • 若块级盒子没有设置宽度，默认宽度和父级一样宽，但是添加浮动后，它的大小根据内容来决定
    • 浮动的盒子中间是没有缝隙的，是紧挨着一起的

1.6浮动元素和标准流搭配使用

先用标准流父元素上下排列位置，之后内部子元素采取浮动排列左右位置。
注意权重问题

2.常见网页布局

2.1常见网页布局

Top——banner——main——footer。

2.2浮动布局注意点

2.2.1浮动和标准流的父盒子搭配

先用标准流父元素上下排列位置，之后子元素采取浮动排列左右位置。

2.2.2一个元素浮动了，理论上其余也要浮动

一个盒子中有多个盒子，有一个盒子浮动了，那么其他的也应浮动。

浮动的盒子只会影响浮动盒子后面的标准流，不会影响前面的标准流。

3.清除浮动

为什么清除浮动

让子盒子撑开父盒子，但盒子浮动不占位置，最后父级盒子高度为0时，就会影响下面标准流的盒子。

3.1清除浮动本质

  • 清楚浮动本质是消除浮动元素造成的影响。
  • 如果父盒子本身有高度，则不需要清除浮动。
  • 清除浮动后，父级就会根据浮动的子盒子自动检测高度，就不会影响标准流了。

3.2清除浮动

语法：

left：不允许左侧有浮动。right：不允许右侧有浮动。both：同时清除左右两侧浮动的影响。

策略是闭合浮动。

3.3清除浮动方法

  1. 额外标签法也成为隔离法。

    会在浮动元素末尾添加一个空的标签。例如：<div style="clear:both"></div>,或者其他标签。新的空标签必须是块级元素。

  2. 父级添加overflow属性。

    给父级添加，属性值写为hidden、auto或scrooll。

  3. 父级添加after伪元素。

    • 优点：没有增加标签，结构更简单

    • 缺点：照顾低版本浏览器

    • 代表网站：百度、淘宝、网易等

  4. 父级添加双伪元素。

    • 优点：代码更简洁

    • 缺点：照顾低版本浏览器

    • 代表网站：小米、腾讯等

3.4CSS属性书写顺序

  1. 布局定位属性：display/position/float/clear/visibility/overflow（display第一个写）
  2. 自身属性：width/height/margin/padding/border/background
  3. 文本属性：color/font/text-decoration/text-align/vertical-align/whit-space/break-word
  4. 其他属性：content/cursor/border-radius/box-shadow/text-shadow/background：linear-gradient…

3.5页面布局整体思路

  1. 确定页面的版心
  2. 分析页面中的行模块，然后每个行模块里的列模块（第一准则）
  3. 一行中列模块经常浮动布局，先确定每个列的大小，之后确定列的位置（第二准则）
  4. 制作HTML结构。遵循现有结构，后有样式的原则。

3.6制作导航栏时注意点

  1. li+a语义更清晰
  2. 不会直接用a标签，而是用li包含链接。若直接用a，搜索引擎容易辨别为有堆砌关键字嫌疑（故意堆砌关键字容易被搜索引擎有降权的风险），从而影响网站排名。
  3. 可以不用给宽度，给一个padding撑开就可以，将来可以继续添加其余文字
  4. 让导航栏一行显示，给li加浮动，需要一行显示

CSS定位

1.定位

1.1为什么需要定位

定位是可以让盒子自由的在某个盒子内移动位置或者固定屏幕中某个位置，并且压住其他盒子。

1.2定位组成

定位：将盒子定在某一个位置，定位也是在摆放盒子，按照定位的方式移动盒子。

定位=定位模式+边偏移

定位模式用于指定一个元素在文档中的定位方式，边偏移决定该元素的最终位置。

1.2.1定位模式

定位模式决定元素定位方式，通过CSS的position属性设置，可以分为四个：

值 语义
static 静态定位
relative 相对定位
absolute 绝对定位
fixed 固定定位
1.2.2边偏移

定位盒子移动最终位置，有top，bottom，left，right四个属性。

边偏移属性 示例 描述
top top:80px 顶端偏移量，定义元素相对于父元素上边线的距离
bottom bottom:80px 底部偏移量，定义元素相对于父元素下边线的距离
left left:80px 左侧偏移量，定义元素相对于父元素左边线的距离
right right:80px 右侧偏移量，定义元素相对于父元素右边线的距离

1.3静态定位static

默认定位方式，无定位的意思

  • 按照标 准流特性摆放，没有偏移。
  • 很少用到

1.4相对定位relative

元素在移动位置时，相对于他原来的位置

特点：

  1. 相对于自己原来的位置移动的，以自己原来的位置为参照物。
  2. 原来标准流的位置继续占有，后面仍然用标准流的方式对待。

1.5绝对定位absolute

绝对定位是元素在移动位置时，相对于祖先元素来说的。

特点：

  1. 如果没有祖先元素或者祖先元素没有定位，以浏览器为准定位。
  2. 如果祖先元素有定位（相对、绝对、固定定位），则以最近一级有定位祖先元素为参考点移动位置。
  3. 不再占有原来的位置（脱标）

1.6子绝父相由来

自己使用绝对定位，父级则需要相对定位。

  1. 子级绝对定位，不会占有位置，可以放到父盒子里面任意一个地方，不会影响其他兄弟盒子。
  2. 父盒子需要加相对定位，让子元素在父元素里。
  3. 父盒子在布局时，需要占有位置，因此父亲只能是相对定位

1.7固定定位fixed

固定定位是固定于浏览器可视区的位置。主要使用于：在浏览器页面滚动时元素的位置不会改变。

特点：

  1. 以浏览器的可视窗口为参照点移动元素。
  2. 和父元素没有任何关系
  3. 不和滚动条滚动
  4. 不占有原先位置。
和版心固定定位
  1. 让固定定位的盒子left：50%，走到浏览器可视区的一半
  2. 让固定定位盒子margin-left版心宽度的一半距离，多走版心宽度的一半位置。

1.8粘性定位sticky

相对定位和固定定位混合

特点：

  1. 以浏览器的可视窗口为参照点移动元素
  2. 粘性定位占有原先的位置
  3. 必须添加top、left、right、bottom其中一个才有效。
定位模式 是否脱标 移动位置 是否常用
static静态定位 否 不能使用边偏移 很少
relative相对定位 否（占有位置） 相对于自身位置移动 常用
absolute绝对定位 是（不占有位置） 带有定位的父级 常用
fixed固定定位 是（不占有位置） 浏览器可视区 常用
sticky粘性定位 否（占有位置） 浏览器可视区 当前阶段很少

1.9定位叠放次序 z-index

会出现盒子重叠，还可以使用z-index控制盒子的前后次序

  • 数值可以是正整数、负数或0，默认是auto，数值越大，盒子越靠上

  • 如果属性相同，后来居上

  • 数字后面不能加单位

  • 只有定位的盒子才可以加

1.10定位的拓展

1.10.1绝对定位盒子居中

加了绝对定位不可以用margin：auto水平居中。

1.10.2定位特殊性
  1. 行内元素添加绝对或者固定定位，可以直接设置高度和宽度
  2. 块级元素添加绝对或者固定定位，如果不给宽度或者高度，默认大小是内容大小。

1.10.3浮动元素、绝对定位不会触发外边距合并问题。

1.10.4绝对定位（固定定位）会压住盒子

浮动元素不同，只会压住下面标准流的盒子，不会压住下面标准流盒子中的文字（图片）

绝对定位会压住标准流下面所有的内容。

让一个元素在页面隐藏或者显示出来

2.1display

属性用于设置一个元素应如何显示，位置不保留

  • display：none；隐藏对象
  • display：block；转换为块级元素，同时显示元素

2.2visibility可见性

指定一个元素可见还是隐藏，位置保留

  • visibility：visible；元素可视
  • visibility：hidden；元素隐藏

scroll 不管是否超出内容，都显示滚动条
auto 超出自动显示，不超出不显示滚动条
